# Тестовое задание — Системный аналитик (Junior)

## Описание

Выполнено тестовое задание на позицию Junior системного аналитика.

В рамках задания были выполнены:
- анализ требований
- проектирование API
- разработка архитектуры системы

## Структура
├── task1_analysis.txt # Анализ требований

---

## Задание 1 — Анализ требований

### Что сделано:
- выявлены противоречия в ТЗ
- найдены неявные и неполные требования
- предложена исправленная версия требований
- сформирован список уточняющих вопросов

### Основные проблемы:
- противоречия в логике корзины (количество, цена)
- неопределённость лимитов
- отсутствие чёткой бизнес-логики
- недостаточная детализация UI и рекламы

---

## Задание 2 — Проектирование API

### Подход:
- REST API
- JSON формат
- версионирование через `/api/v1/`

### Реализовано:
- endpoint для получения списка партнёров
- поддержка пагинации (limit/offset)
- единый формат ответа
- обработка ошибок
- описание структуры данных

### Особенности:
- идемпотентность GET-запросов
- расширяемость (новые типы доставки)
- разделение data и meta

---

## Задание 3 — Архитектура

### Архитектурный стиль:
- клиент-сервер
- микросервисный подход

### Компоненты:
- Mobile App
- API Gateway
- Partner Service
- Cart Service
- Notification Service
- Database
- Message Broker
- Push Provider (Firebase / APNS)

### Ключевые решения:
- разделение ответственности между сервисами
- использование API Gateway как единой точки входа
- асинхронная обработка через Message Broker
- изоляция Notification Service

---

## Основные принципы

- масштабируемость
- расширяемость
- слабая связанность сервисов
- единый контракт API
- предсказуемое поведение системы
