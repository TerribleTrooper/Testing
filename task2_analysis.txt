ЗАДАНИЕ 1. АНАЛИЗ ТРЕБОВАНИЙ

========================================
1. ВЫЯВЛЕННЫЕ ПРОТИВОРЕЧИЯ И НЕДОЧЁТЫ
========================================

1. Противоречие (п.2 и п.9)
Проблема:
Пункт 2 запрещает уменьшать количество товара меньше 1.
Пункт 9 допускает уменьшение до 0 с последующим удалением.

Почему это проблема:
Невозможно реализовать оба сценария одновременно.

Вывод:
Требования к минимальному количеству противоречат сценарию удаления.


2. Противоречие (п.7 и п.13)
Проблема:
Пункт 7 — цена фиксируется и не меняется.
Пункт 13 — цена обновляется при изменении в каталоге.

Почему это проблема:
Нельзя одновременно использовать фиксированную и динамическую цену.

Вывод:
Не определено поведение цены (snapshot или динамическая).


3. Неопределённость лимитов (п.1, п.3, п.4)
Проблема:
Есть несколько ограничений:
- до 10 единиц одного товара
- до 5 разных товаров
- до 20 единиц суммарно

Не указан приоритет лимитов.

Почему это проблема:
Разные реализации приведут к разному поведению системы.


4. Поведение при превышении лимитов (п.6)
Проблема:
Указано сообщение "Лимит корзины превышен", но:
- не указано какой лимит
- не описано поведение системы (отмена, частичное добавление)

Почему это проблема:
Невозможно корректно реализовать UX и бизнес-логику.


5. Избыточный пункт (п.5)
Проблема:
"Товары в корзине могут быть разные" — очевидно и дублирует п.3.


6. Реклама (п.10 и п.11)
Проблема:
- не указано место отображения
- не указано количество блоков
- "утром и вечером" не имеет точного времени

Почему это проблема:
Невозможно реализовать без дополнительных уточнений.


7. Недостаточная детализация UI (п.8)
Проблема:
Не указаны:
- формат отображения
- валюта
- правила округления
- сортировка

Почему это проблема:
Разработчики и дизайнеры могут реализовать по-разному.


8. Два способа удаления (п.2 и п.9)
Проблема:
Удаление возможно:
- через кнопку
- через уменьшение до 0

Не указано, допустимы ли оба варианта.


========================================
2. ИСПРАВЛЕННАЯ ВЕРСИЯ ТЗ
========================================

Функционал корзины:

1. Пользователь может добавить в корзину от 1 до 10 единиц одного товара.

2. Пользователь может изменять количество товара в диапазоне от 1 до 10.

3. Удаление товара осуществляется:
   - через кнопку "Удалить"
   - либо при установке количества = 0

4. В корзине может находиться:
   - не более 5 различных товаров
   - не более 20 единиц товаров суммарно

5. При попытке превышения лимитов:
   - операция не выполняется
   - отображается сообщение:
     - "Превышено максимальное количество товаров"
     - "Превышено максимальное количество единиц"

6. Цена товара:
   - фиксируется в момент добавления в корзину
   - не изменяется при изменении цены в каталоге

7. В корзине отображается:
   - список товаров
   - название товара
   - количество
   - цена за единицу
   - итоговая стоимость по позиции

8. Валюта отображения: RUB

9. Реклама:
   - отображается внизу корзины
   - не более 1 рекламного блока
   - показывается с 08:00 до 12:00 и с 18:00 до 22:00

ЗАДАНИЕ 2. ПРОЕКТИРОВАНИЕ API
========================================

1. ОБЩЕЕ ОПИСАНИЕ

API предназначено для получения списка партнеров с информацией о доставке.

Формат взаимодействия: REST
Формат данных: JSON
Версионирование: через URL (/api/v1/)

Базовый URL:
https://api.example.com


2. ENDPOINT

GET /api/v1/partners

Описание:
Возвращает список доступных партнеров.


3. QUERY ПАРАМЕТРЫ

cityId (integer, optional)
Идентификатор города

limit (integer, optional, default = 10)
Количество записей на страницу

offset (integer, optional, default = 0)
Смещение для пагинации


4. RESPONSE

HTTP 200 OK

{
  "data": [
    {
      "id": 1,
      "name": "METRO",
      "logoUrl": "https://cdn.app/metro.png",
      "delivery": {
        "type": "slot",
        "label": "Ближайшая доставка сегодня",
        "from": "21:00",
        "to": "23:00"
      },
      "redirectUrl": "https://metro.ru"
    }
  ],
  "meta": {
    "total": 25,
    "limit": 10,
    "offset": 0
  }
}


5. СТАТУС-КОДЫ

200 OK — успешно  
400 Bad Request — ошибка запроса  
404 Not Found — не найдено  
500 Internal Server Error — ошибка сервера
